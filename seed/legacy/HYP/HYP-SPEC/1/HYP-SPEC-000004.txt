# HYP-SPEC-000005

# Terminology

**Project:** Hypatia Knowledge Platform

**Document ID:** HYP-SPEC-000005

**Version:** 1.0

**Status:** Foundation Document

**Classification:** Official Terminology

---

# Purpose

This document defines the official vocabulary used throughout the Hypatia project.

Every technical document, database schema, source code, JSON specification, API, and research paper produced by Hypatia must use these terms consistently.

No synonym should replace an official term.

---

# A

## Artifact

Any historical object carrying information.

Examples

* Papyrus
* Stela
* Ostracon
* Statue
* Wall inscription
* Tablet

---

# B

## Builder

The subsystem responsible for constructing structured knowledge from raw data.

Examples

Hypatia Builder

Training Builder

Knowledge Graph Builder

Dataset Builder

---

# C

## Canonical

The officially accepted version of an object after academic validation.

Canonical does **not** mean absolute truth.

It represents the best supported scholarly interpretation available.

---

## Canonical Form

The normalized scientific representation of an object.

Examples

Canonical Lemma

Canonical Transliteration

Canonical Entity Name

---

## Card

The smallest independent knowledge object stored by Hypatia.

Every Card has:

* ID
* Metadata
* Relationships
* Version History

Examples

Lemma Card

Entity Card

Story Card

Grammar Card

Source Card

---

## Community

The body of researchers contributing proposals and reviews.

The community improves knowledge.

It never owns knowledge.

---

## Corpus

A structured collection of texts.

A Corpus may contain:

Stories

Documents

Letters

Medical texts

Mathematical texts

Administrative texts

A Corpus is **not** a database.

It is a scholarly collection.

---

# D

## Dataset

Machine-readable data produced for computational purposes.

Examples

Training Dataset

Validation Dataset

Inference Dataset

Evaluation Dataset

Datasets are generated.

They are never manually edited.

---

## Domain

A scientific classification describing the field of knowledge.

Examples

Religion

Medicine

Astronomy

Administration

Military

Literature

Architecture

Law

---

# E

## Entity

Any identifiable named object.

Examples

King

Queen

Person

God

City

Temple

Dynasty

Festival

Ship

Document

Institution

Every Entity possesses one permanent identifier.

---

## Evidence

A verifiable source supporting a statement.

Every scholarly claim must reference evidence.

---

# G

## Grammar

The formal linguistic rules governing the language.

Grammar includes:

Morphology

Syntax

Orthography

Writing conventions

Grammar Cards are independent knowledge objects.

---

# K

## Knowledge Base

The complete structured repository of Hypatia.

It consists of all validated knowledge objects and their relationships.

---

## Knowledge Graph

The network created by connecting knowledge objects through explicit relationships.

The Knowledge Graph is generated from the repository.

It is not the repository itself.

---

# L

## Layer

A logical processing stage inside Hypatia.

Examples

Source Layer

Knowledge Layer

Relationship Layer

Training Layer

Presentation Layer

Each Layer depends only on lower layers.

---

## Lemma

The canonical lexical identity of a word.

The Lemma is the primary linguistic anchor of the repository.

Translations may change.

Spellings may vary.

Occurrences may differ.

The Lemma remains constant.

---

## Lexicon

The complete collection of Lemmas.

---

# M

## Meta

Structured descriptive information about an object.

Meta never contains the object itself.

Meta describes the object.

Examples

Creation Date

Author

Language

Historical Period

Discovery Location

Confidence

---

## Metadata

The collection of Meta fields describing an object.

---

# O

## Occurrence

A specific appearance of a Lemma inside a Story.

One Lemma may have thousands of Occurrences.

Occurrences are contextual.

Lemmas are lexical.

---

# P

## Proposal

A suggested modification submitted by a researcher.

A Proposal never changes the repository directly.

It enters the Review System.

---

# R

## Relationship

An explicit scientific connection between two objects.

Examples

Lemma → Story

Entity → Story

Story → Source

Entity → Entity

Lemma → Grammar Rule

Relationships are stored independently.

---

## Repository

The authoritative storage system of Hypatia.

Everything else consumes the Repository.

---

## Resolver

A subsystem responsible for identifying, linking, or normalizing knowledge objects.

Resolvers never invent knowledge.

Resolvers only resolve.

Examples

Lemma Resolver

Entity Resolver

Grammar Resolver

Domain Resolver

Source Resolver

---

## Review

The scientific evaluation process applied to a Proposal.

---

## Reviewer

An authorized academic responsible for validating proposals.

---

## Revision

A preserved historical version of an object.

No Revision is deleted.

---

# S

## Sense

A distinct meaning of a Lemma.

One Lemma may possess multiple Senses.

Each Sense may contain:

Definition

Translation

Examples

Confidence

Supporting Sources

---

## Source

The origin of knowledge.

Examples

Papyrus

Journal

Book

Museum

Excavation Report

Dictionary

Research Article

---

## Story

A complete textual document.

Examples

Letter

Papyrus

Stela

Administrative Record

Literary Composition

Medical Treatise

Mathematical Text

A Story contains many Occurrences.

---

# T

## Token

A single lexical unit extracted from text.

Tokens belong to Stories.

Tokens are linked to Lemmas.

---

## Transliteration

The scholarly phonetic representation of an ancient script.

Transliteration is language-dependent.

It is not the Lemma.

---

# V

## Validation

The process of confirming scholarly correctness before Canonical acceptance.

---

## Version

A numbered state of an object.

Versions allow complete historical reconstruction.

---

# W

## Workspace

A temporary environment used before official publication.

Objects inside a Workspace are not yet Canonical.

---

# Reserved Words

The following terms are reserved by the Hypatia platform and must never be reused with different meanings.

Lemma

Entity

Story

Source

Sense

Occurrence

Resolver

Proposal

Review

Revision

Knowledge Graph

Canonical

Meta

Domain

Corpus

Builder

Repository

Workspace

Version

---

# Naming Rule

Every future specification, JSON schema, database table, Python class, or API endpoint must use these exact terms.

No aliases.

No abbreviations.

No translations inside the codebase.

Consistency is considered part of scientific accuracy.

---

# Closing Statement

A shared language produces shared understanding.

A shared understanding produces consistent knowledge.

Consistent knowledge produces reliable intelligence.

Therefore,

The official terminology of Hypatia is considered part of the scientific infrastructure of the project.
