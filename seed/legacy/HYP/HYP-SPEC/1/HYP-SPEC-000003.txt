# Project Scope

**Project:** Hypatia Knowledge Platform

**Version:** 1.0

**Status:** Foundation Document

**Classification:** Core Specification

---

# 1. Purpose

This document defines the official scope of the Hypatia Knowledge Platform.

It specifies what belongs to Hypatia.

It also specifies what does **not** belong to Hypatia.

Maintaining this boundary is essential to preserve architectural clarity and scientific consistency.

---

# 2. Definition

Hypatia is a structured Knowledge Platform.

Its primary purpose is to preserve, organize, validate, relate, and expose knowledge about ancient civilizations in a machine-readable and human-readable form.

Hypatia is **not** designed as an application.

Applications are consumers of Hypatia.

---

# 3. Primary Domain

The first supported civilization is:

**Ancient Egypt**

Primary writing systems:

* Hieroglyphic
* Hieratic
* Demotic
* Coptic (future stage)

---

# 4. Future Domains

The architecture intentionally supports future civilizations.

Examples include:

* Mesopotamia
* Sumer
* Akkad
* Babylon
* Assyria
* Hittites
* Phoenicia
* Ancient Greece
* Ancient Rome

No architectural redesign should be required to support future civilizations.

# 9. What Hypatia Will Never Store

Hypatia intentionally excludes:

Executable programs

Compiled models

Flutter applications

NodeJS services

Temporary cache

Logs

Binary AI weights

Generated responses

Runtime sessions

User preferences

These belong to software repositories.

Not to the knowledge repository.

---

# 10. Scientific Boundary

Hypatia stores knowledge.

It does not publish conclusions.

It does not create historical theories.

It does not resolve scholarly debates.

It documents evidence.

---

# 11. AI Boundary

Artificial Intelligence is a consumer.

It may:

Search.

Summarize.

Translate.

Infer.

Predict.

Suggest.

It may never overwrite the repository.

---

# 12. Community Boundary

Researchers may propose.

Editors may validate.

Reviewers may approve.

Only accepted knowledge becomes canonical.

---

# 13. Scalability Boundary

Every object added to Hypatia must satisfy three conditions.

It must:

be identifiable

be versioned

be relational

If an object cannot satisfy these three conditions,

it does not belong to Hypatia.

---

# 14. Canonical Principle

Every scientific fact stored inside Hypatia must have exactly one canonical identity.

That identity never changes.

Its interpretation may evolve.

Its identifier never does.

---

# 15. Repository Goal

The repository is expected to become the authoritative structured knowledge source from which:

Websites are built.

Search engines operate.

Researchers work.

Artificial Intelligence learns.

Future generations preserve knowledge.

---

# 16. Scope Statement

Hypatia does not aim to become the largest collection of ancient texts.

It aims to become the **best organized scientific knowledge repository** for ancient civilizations.

Quality takes precedence over quantity.

Structure takes precedence over accumulation.

Relationships take precedence over isolated records.

---

# 17. Scope Motto

> **If knowledge cannot be connected, it is not yet complete.**
