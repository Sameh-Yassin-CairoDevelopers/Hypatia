# HYP-SPEC-010300

# File Naming Convention

**Project:** Hypatia Knowledge Platform

**Document ID:** HYP-SPEC-010300

**Version:** 1.0 Founder Edition

**Status:** Official

**Classification:** Architecture Standard

---

# Purpose

This specification defines the official naming convention for every file stored inside the Hypatia Repository.

Every filename shall be:

* Stable
* Predictable
* Machine-readable
* Human-readable
* Scientifically meaningful

---

# Design Philosophy

A filename is part of the knowledge.

It is not merely an operating system label.

A filename must immediately reveal:

* object type
* object identity
* scientific class

without opening the file.

---

# General Rules

Every filename shall be

* lowercase
* ASCII only
* immutable
* unique
* deterministic

---

# Founder Statement

A disciplined repository begins with disciplined names.

If filenames remain stable,

the repository remains understandable,

regardless of how large it becomes.

---

# Academic Notes

The naming convention intentionally separates **Identity** from **Semantics**.

A Lemma may change.

A Story title may change.

An Entity description may change.

Scientific interpretation may evolve.

But identifiers—and therefore filenames—remain immutable.

This guarantees stable citations, reproducible research, long-term archival integrity, and compatibility with future database engines and AI training pipelines.

## ✦ ملاحظة أخيرة (وأعتبرها من أهم قرارات المشروع)

أثناء كتابة هذه الوثيقة، وصلت إلى قناعة أن **هيباتيا يجب ألا تستخدم أسماء الملفات كمعرّفات**.

بل يكون لدينا دائمًا:

* **ID داخلي دائم** داخل الملف (مثل `LEM-000001`)
* **اسم ملف قياسي** مشتق منه (مثل `lem_000001.json`)

أي أن الملف مجرد وعاء، أما الهوية الحقيقية فهي الحقل `id` داخل الـ JSON.

هذا سيجعل النظام أكثر صلابة إذا احتجنا مستقبلًا إلى نقل البيانات من ملفات JSON إلى قاعدة رسوم بيانية أو قاعدة علائقية أو أي وسيط تخزين آخر دون أن تتغير هوية أي عنصر. وأنا أوصي بأن يصبح هذا مبدأ ثابتًا في هيباتيا.
