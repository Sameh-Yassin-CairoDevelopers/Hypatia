# HYP-SPEC-010400

# ID Convention

**Project:** Hypatia Knowledge Platform

**Document ID:** HYP-SPEC-010400

**Version:** 1.0 Founder Edition

**Status:** Official

**Classification:** Architecture Standard

---

# Purpose

This specification defines the official identity system used throughout the Hypatia Platform.

Every object stored by Hypatia must possess exactly one permanent identifier.

The identifier is immutable.

The identifier is globally unique.

The identifier survives all future revisions.

---

# Fundamental Principle

Identity never changes.

Knowledge evolves.

Identity does not.

---

# Identity Hierarchy

Every object is identified by three levels.

```text
Platform

↓

Object Type

↓

Object Number
```

Example

```text
HYP-LEM-000001
```

# ID Structure

The official structure is

```text
AAA-TTT-NNNNNN
```

Where

AAA

Project Prefix

TTT

Object Prefix

NNNNNN

Sequential Identifier

---

```text

```

# Academic Notes

The Hypatia ID system deliberately separates **Scientific Identity** from **Technical Identity**.

* **Scientific Identity (`HYP-LEM-000145`)** is permanent, citable, readable by humans, and appears in documentation, publications, APIs, and relationships.
* **Technical Identity (`UUID`)** exists only to optimize storage engines and database performance.

This separation ensures that Hypatia can migrate between JSON files, relational databases, graph databases, document stores, or future storage technologies **without ever changing the scholarly identity of a single object**.
