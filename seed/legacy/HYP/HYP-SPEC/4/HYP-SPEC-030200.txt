**Project:** Hypatia Knowledge Platform

**Classification:** Core Pipeline

# Purpose

The Normalization Pipeline transforms imported data into the canonical internal representation used throughout the Hypatia Repository.

It establishes a unified language for all knowledge regardless of its original source.

---

# Philosophy

Every source speaks its own language.

Hypatia speaks one language.

Normalization creates that language.

---

# Responsibilities

The Normalization Pipeline is responsible for

* Field normalization
* Identifier normalization
* Encoding normalization
* Language normalization
* Structure normalization
* Metadata normalization
* Reference normalization

It never modifies scientific meaning.

## **Canonical Mapping Registry**

ويكون ملفًا مستقلًا داخل المشروع:

```text
repository/

normalization/

canonical_mapping_registry.json
```

ويحتوى على جميع خرائط التحويل بين المصادر المختلفة والبنية القياسية لهيباتيا.

مثال:

```text
TLA
translation
↓

meaning_en

PharaLex
reading
↓

transliteration

BA
glyph
↓

hieroglyphs
```

ولا يقتصر على أسماء الحقول فقط، بل يشمل أيضًا:

* قيم التعدادات (Enums)
* أكواد اللغات
* أنواع الكيانات
* أنواع الوثائق
* الاختصارات

بهذا يصبح دعم مصدر جديد عملية إضافة **خريطة تحويل (Mapping)** فقط، دون تعديل أي سطر في الكود أو في المستودع نفسه. وأرى أن هذا القرار سيجعل هيباتيا قابلة للتوسع لعشرات السنين مع الحفاظ على ثبات معمارية المشروع.
