# Purpose

The Training Dataset Builder transforms the validated scientific knowledge stored inside the Hypatia Repository into structured datasets suitable for training Artificial Intelligence models.

The generated datasets preserve scientific provenance, reproducibility, and explainability.

---

# Philosophy

The Repository is the source of truth.

Training datasets are reproducible products generated from the Repository.

No dataset is edited manually.

Every dataset can be regenerated at any time.

---

# Responsibilities

The Training Dataset Builder is responsible for

* Selecting eligible Knowledge Objects.
* Generating supervised training samples.
* Preserving scientific references.
* Producing task-specific datasets.
* Recording dataset metadata.
* Versioning every generated dataset.

It never modifies Repository Objects.

# Founder Statement

Artificial Intelligence should never become the primary keeper of knowledge.

Knowledge must remain independent, permanent, and scientifically verifiable.

The Training Dataset Builder ensures that every future AI model learns from a transparent, reproducible, and evidence-based representation of the Hypatia Repository.

As the Repository grows, every future model automatically benefits without requiring a redesign of the underlying scientific archive.

---

# Academic Notes

The Training Dataset Builder establishes a strict separation between scientific knowledge and machine learning artifacts.

Repository Objects remain immutable scholarly records, while datasets are transient computational products derived from them.

This architecture guarantees repeatable experiments, long-term reproducibility, and the ability to regenerate identical datasets from any historical Repository revision, making Hypatia suitable for both academic research and industrial-scale AI development.
