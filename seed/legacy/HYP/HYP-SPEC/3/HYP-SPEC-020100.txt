# Purpose

This specification defines the Meta Card.

A Meta Card represents any reusable concept used to classify, describe, normalize, organize, or connect Knowledge Objects.

Meta Cards are first-class Knowledge Objects.

They are not auxiliary metadata.

---

# Philosophy

Everything that repeats should become Meta.

Every classification should become Meta.

Every controlled vocabulary should become Meta.

Knowledge should never duplicate descriptive information.

---

# Definition

A Meta Card represents a reusable scientific concept.

It does not describe one object.

It describes many objects.

---

Example

Instead of

```text id="v1"
Language = Middle Egyptian
```

written inside 500,000 cards

Hypatia stores

```text id="v2"
META-LANGUAGE-000001

Middle Egyptian
```

Then every object references it.

# Founder Statement

Meta Cards are the vocabulary through which Hypatia understands its own knowledge.

By elevating classifications into independent Knowledge Objects, the platform eliminates duplication, strengthens consistency, and allows every future addition to connect naturally with the existing knowledge graph.

---

# Academic Notes

Unlike traditional systems where metadata is merely attached to records, Hypatia treats Meta as a **scientific entity with its own identity, history, relationships, and evolution**.

This design transforms metadata into a reusable semantic layer that supports:

* consistent classification,
* ontology construction,
* multilingual normalization,
* resolver accuracy,
* knowledge graph expansion,
* and future AI reasoning.

The Meta Layer is therefore not an accessory to the repository—it is one of the foundational pillars upon which the entire Hypatia ecosystem is built.
