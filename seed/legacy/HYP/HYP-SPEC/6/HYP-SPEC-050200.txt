# Purpose

The Corpus Builder transforms the scientifically validated Repository into structured corpora suitable for Artificial Intelligence training.

Its responsibility is not merely collecting texts, but constructing a coherent, balanced, traceable, and reproducible linguistic representation of Ancient Egyptian civilization.

The Corpus is therefore considered a derived scientific artifact generated from the Repository.

---

# Philosophy

The Repository stores knowledge.

The Corpus organizes knowledge for learning.

The Model learns from the Corpus.

The Repository remains unchanged.

---

# Fundamental Principle

The Corpus is always generated.

It is never edited manually.

Whenever the Repository changes, the Corpus may be regenerated.

---

# Corpus Sources

The Corpus Builder consumes only Repository Objects.

Possible sources include

* Story Cards
* Dictionary Cards
* Grammar Cards
* Entity Cards
* Relation Cards
* Domain Cards
* Source Cards

No external dataset enters the Corpus directly.

External data must first become Repository knowledge.

```text

```

# Canonical Representation

Each Corpus Entry is stored using its canonical representation.

Canonical data includes

* canonical transliteration
* canonical hierarchy
* canonical identifiers
* canonical metadata

This guarantees consistency across all future training runs.

---

# Corpus Diversity

The Builder maintains balance across

* historical periods
* writing traditions
* text genres
* domains
* linguistic complexity

No single category should dominate the Corpus unless scientifically justified.

---

# Corpus Integrity

* conflicting entries

# Academic Notes

The Hypatia Corpus Builder implements a **Repository-Derived Corpus Architecture (RDCA)**.

Unlike conventional corpora assembled directly from heterogeneous textual sources, the Hypatia Corpus is deterministically generated from validated Repository Objects.

This architecture establishes complete traceability between every training example and its scholarly origin, enabling exact reproducibility, systematic quality control, controlled historical evolution, and seamless regeneration following Repository revisions.
