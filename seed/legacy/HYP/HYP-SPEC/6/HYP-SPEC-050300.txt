# Purpose

This document defines the philosophy governing tokenization within the Hypatia ecosystem.

Unlike conventional language models, the Hypatia tokenizer is designed to preserve the linguistic, semantic, grammatical, and historical structure of Ancient Egyptian writing rather than merely compressing text into computational units.

Its objective is to maximize scientific meaning before computational efficiency.

---

# Philosophy

The tokenizer is not a compression algorithm.

It is a linguistic interpreter.

Its responsibility is to preserve meaning while preparing knowledge for machine learning.

---

# Fundamental Principle

Meaning is always more important than token count.

Token efficiency must never destroy linguistic structure.

---

# Scientific Position

The tokenizer is derived from the Repository.

It is never learned directly from raw text.

Its vocabulary is governed by scientific knowledge rather than statistical frequency.

---

The tokenizer always follows the Repository.

# Linguistic Philosophy

Ancient Egyptian writing contains multiple linguistic layers.

The tokenizer should preserve them whenever possible.

Examples include

* Sign
* Biliteral
* Triliteral
* Determinative
* Phonetic Complement
* Word
* Compound Expression
* Formula
* Cartouche
* Proper Name

These layers are scientific objects rather than arbitrary symbols.

---

↓

Training may operate at different hierarchy levels depending on the task.

---

# Canonical First

Every token originates from the canonical representation stored inside the Repository.

Tokenization never invents alternative spellings.

Canonical knowledge always has priority.

---

# Stable Vocabulary

Vocabulary evolves only when the Repository evolves.

Training frequency never changes vocabulary structure.

Vocabulary revisions are Repository revisions.

---

# Scientific Tokens

Vocabulary includes scientific entities unavailable in ordinary tokenizers.

Examples

* Hieroglyphic signs
* Transliteration units
* Lemmas
* Grammar markers
* Entity identifiers
* Domain identifiers

These are first-class linguistic objects.

---

# Lemma Awareness

Whenever possible,

the tokenizer preserves the relationship between

Word

↓

This allows later models to reason beyond surface forms.

---

# Grammar Awareness

The tokenizer preserves grammatical information whenever available.

Examples

* verb class
* gender
* number
* tense
* person
* state

Grammar remains linked to Repository Objects.

---

# Entity Awareness

Named entities remain identifiable.

Kings,

Places,

Gods,

Temples,

Dynasties

should never lose their identity during tokenization.

---

# Context Preservation

Token boundaries should preserve semantic context rather than maximizing compression.

Breaking meaningful expressions into statistically convenient fragments is discouraged.

---

# Explainability

Every produced token must be traceable.

Given a token,

the Repository should be capable of explaining

* origin
* canonical object
* linguistic meaning
* Repository revision

---

# Vocabulary Versioning

Every tokenizer generation records

```text id="tokenizer_version"
tokenizer_revision

repository_revision

builder_version

generation_date
```

Training remains reproducible.

---

# Backward Compatibility

Vocabulary evolution should preserve compatibility whenever possible.

Existing trained models should remain usable across minor tokenizer revisions.

Breaking changes require explicit major versions.

---

# AI Position

The tokenizer serves the Repository.

It does not optimize itself independently.

Its objective is scientific consistency rather than statistical compression.

# Long-Term Objective

To create one of the first repository-driven tokenization systems specifically designed for Ancient Egyptian language, capable of preserving linguistic structure, historical semantics, grammatical relationships, and scholarly traceability across future generations of Artificial Intelligence.

Knowledge defines tokens.

# Founder Statement

For most language models, the tokenizer is an engineering component.

For Hypatia, it is a linguistic instrument.

Every decision taken during tokenization affects how the model perceives an entire civilization.

Therefore, tokenization is treated as an extension of Egyptological scholarship rather than a preprocessing step.

The tokenizer must preserve as much scientific meaning as possible before a single parameter of the model begins to learn.

---

# Academic Notes

The Hypatia Tokenizer adopts a **Knowledge-Guided Tokenization (KGT)** paradigm.

Rather than deriving vocabulary exclusively from statistical frequency, vocabulary construction is constrained by Repository-defined linguistic objects, canonical representations, lemma mappings, grammatical metadata, and entity identities.

This approach reduces semantic fragmentation, improves explainability, enables symbolic integration, and establishes direct traceability between neural representations and the underlying scholarly knowledge base.
